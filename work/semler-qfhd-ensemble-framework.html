Problem

Model ensemble experiments generate CSV output files across multiple folds and checkpoints. These files needed centralized processing and storage for downstream analysis and frequency tracking, with a scalable, serverless infrastructure that could handle variable workloads without manual intervention.

Contribution

Built a complete serverless data pipeline to ingest, process, and index ensemble CSV outputs. Designed infrastructure-as-code using Terraform for persistent resources (S3, DynamoDB, IAM) and Serverless Framework for compute (Lambda, Step Functions). Implemented CSV parsing and validation in Lambda, DynamoDB schema modeling across two tables (FileMetadata, ExperimentPredictions), and Step Function state machines for distributed batch processing with automatic retries and concurrency control.

Technical details

Architecture: S3 triggers Lambda on CSV upload → CSV processor extracts fold/model/split metadata from object key → writes predictions to ExperimentPredictions table, metadata to FileMetadata table. Separate frequency-calculator Lambda computes file occurrence counts. Step Function orchestrates batch scans of FileMetadata using pagination and concurrent Lambda invocations with configurable batch size and retry logic.

Key refactors: Migrated from monolithic Terraform to modular Serverless Framework with serverless-compose for service separation (Lambda, Step Functions, IAM, CloudWatch). Split single DynamoDB table into FileMetadata (partition: Filename, sort: Frequency) and ExperimentPredictions (partition: Filename, sort: ExperimentID#SplitType) for cleaner queries. Removed serverless.yml at project root; consolidated Serverless config under infrastructure/serverless with cross-service references via Fn::ImportValue. Simplified Step Function state machine by replacing redundant scanning states with recursive pagination using ExclusiveStartKey.

Observability: CloudWatch log groups per service with retention policies. Enhanced error propagation by removing error-handling states and raising exceptions instead.

Outcome

Deployed to production and handed off to the a medical-device manufacturer team. The pipeline supports multi-modal ensemble inference — combining predictions from models trained on cardiac signals alone and signals + spectrogram representations — aggregating fold-level outputs at scale. Frequency tracking enables downstream analysis on model agreement and experiment coverage across folds.
