Problem

Llama.cpp runs LLM inference across diverse hardware platforms using specialized compute backends (Metal, CUDA, Vulkan, etc.). WebGPU — a modern, platform-agnostic GPU abstraction — was missing from this ecosystem, limiting inference to browser and cross-platform edge environments where only CPU compute was practical.

Contribution

Implemented WebGPU backend support for llama.cpp, advancing from initial unary operators to a mature, production-ready backend covering binary operators, critical compute kernels, and optimized JIT compilation. Ported six unary operators (FLOOR, CEIL, ROUND, TRUNC, EXPM1, SOFTPLUS) in 2025, then expanded through 2026 to include JIT-compiled binary operators (with automatic overlap handling), mul_mat variants (reg_tile, subgroup_matrix, standard), copy, GLU, RoPE, and soft_max operations.

Technical details

WebGPU backend integrates with GGML's compute abstraction as a first-class backend (alongside Metal, CUDA, Vulkan). Core work focused on two architectural shifts:

2025: Fixed xielu parameter passing (merged to fork), added initial unary operator implementations.

2026 Q1: Migrated binary operators from ahead-of-time (AOT) template expansion to just-in-time (JIT) pre-WGSL compilation with automatic overlap detection (distinguishing in-place, source/destination overlap, and default bindings). Resolved WebGPU's 65,535 workgroup-per-dimension dispatch limit for large batch sizes by implementing 2D workgroup provisioning and bounds checking across mul_mat kernels. Ported all remaining AOT operators to JIT compilation by end of Q1, consolidating shader generation into compile-time preprocessing and eliminating Python template expansion.

Outcome

WebGPU backend is now a fully-supported, in-progress backend in llama.cpp. Matured from proof-of-concept (unary operators) to production-ready pipeline: JIT shader compilation, multi-kernel support (binary, mul_mat variants, critical elementwise operations), scalable batch handling, and binding correctness. This enables GPU-accelerated LLM inference on browsers, cross-platform devices, and edge hardware without native CUDA, HIP, or Metal support.
