Problem

Hospitals preserve all endoscopy procedure videos as training material for AI models. However, these videos require expert retrospective labeling to be useful—a laborious, offline process that introduces accuracy drift (labels written hours or days after observation, without real-time clinical context).

Contribution

Built eusml-labeller, a web application enabling experts to label endoscopic ultrasound videos during the procedure. The platform captures patient identifiers (name, MRD), morphological findings (Normal/CCP/Cysts/Other), procedure locations (Stomach/D1/D2), multiple examination stations, and FNA (fine-needle aspiration) events with precise timing.

Technical details

Implemented as a Remix full-stack application deployed on AWS with GitHub Actions CI/CD. Frontend includes interactive stopwatches for station-level timing, database-backed storage for patient/procedure/station records, and real-time JSON output for labeling verification. Built form validation for patient data, responsive UI with mobile-friendly table scrolling, and state management to enforce procedural constraints (e.g., only one station active at a time unless FNA).

Outcome

Shipped in September 2023 and continued through June 2024. In 2024, added a Tumor Timer enabling real-time labeling of tumor sightings during the procedure, and added patient data encryption with restricted decryption to meet clinical data compliance requirements. Application enables in-situ labeling, eliminating retrospective annotation delay and improving label accuracy through real-time clinical context.
