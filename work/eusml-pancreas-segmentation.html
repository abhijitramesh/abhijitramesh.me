Problem

Endoscopic ultrasound (EUS) data collection for pancreas segmentation required a scalable dataset generation pipeline. Raw video data needed preprocessing—frame extraction, optimal sampling, and metadata tracking—before annotation. Integration with annotation tools like CVAT required careful frame selection to balance data density with annotator workload.

Contribution

Built the complete ML dataset pipeline from first commit to production use. Designed and implemented video frame extraction using AV (ffmpeg bindings), manifest generation for dataset tracking, and DynamoDB integration for patient metadata. Optimized frame sampling strategy through iterative refinement: initial whole-video extraction → FPS subsampling → 25-frame-per-video strategy, reducing annotation overhead while preserving segmentation signal.

Technical details

Set up project infrastructure with Poetry (dependency management) and Hydra (configuration). Implemented video processing pipeline using AV for efficient frame extraction and ffmpeg-level transformations (cropping, resampling). Created manifest generation to track dataset lineage and versioning. Connected to DynamoDB for metadata queries (patient records, study IDs). Integrated frame sampling logic to feed annotated data into CVAT, iteratively tuning sample rates based on annotation feedback.

Outcome

Delivered a production-ready dataset generation pipeline processing EUS videos for pancreas segmentation. Pipeline handles end-to-end flow: raw video ingestion, intelligent frame sampling, metadata enrichment, and annotation tool integration. Active development extended through late 2025 with ongoing refinements to sampling strategies and pipeline robustness.
