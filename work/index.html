Selected work

Where I’ve shipped.

Four chapters: WebGPU systems work at UCSC, healthcare AI at Theta Tech AI and TexNano, and the open-source thread that runs through all of it. Numbers are real, links go to public artifacts.

2025 → present · Graduate Researcher

UCSC — Concurrency and Heterogeneous Programming Lab

With Dr. Tyler Sorensen and PhD student Reese Levine. Contributing the WebGPU backend to ggml-org/llama.cpp so LLM inference runs in any browser with a GPU.

  • 3 PRs merged to ggml-org/llama.cpp in Q1 2026 — spanning shader compilation, kernel correctness, and dispatch-limit fixes (#19310, #19965, #20728).
  • Migrated binary operators from AOT template expansion to JIT pre-WGSL compilation, with automatic source/destination overlap detection and binding correctness across in-place and default cases. Eliminated the Python template-expansion path entirely.
  • Fixed WebGPU’s 65,535 workgroup-per-dimension dispatch limit for large batch matrix-multiplies — introduced a 2D workgroup-provisioning helper and bounds checks across every mul_mat kernel variant (reg_tile, subgroup_matrix, standard).
  • Ported cpy, glu, rope, and soft_max shaders from AOT to JIT, completing the migration.
  • Built webgpu-bench — an end-to-end benchmark that compiles llama.cpp to WebAssembly and drives Chrome and Safari via WebGPU. Validates numerical correctness via forced-decoding consistency. Tracks 10 models across 194 quantization variants. Live dashboard at webgpu-bench.abhijitramesh.me.
  • Isolated and filed a Q4_K kernel bug in Dawn’s Metal backend on Apple Silicon — minimal browser and native C++ repro upstream, with two known toggle-fixes documented.

2022 → 2025 · ML Engineer

Theta Tech AI — Semler Scientific / QuantaFlo

QuantaFlo is an FDA-pathway medical device that detects Peripheral Arterial Disease from infrared blood-volume waveforms. Joined as data and modeling lead; stayed for the production push.

  • Built the QuantaFlo data and training pipeline from zero in 2022 — signal preprocessing, self-supervised model research, Optuna hyperparameter search, Neptune experiment tracking, a Dash labeling app on DynamoDB, and a Lambda → S3 → EC2 inference pipeline on AWS. Project shipped.
  • 171 commits in 2023 transitioning the pipeline from research to production MLOps: Lambda-triggered Step Functions, SageMaker Docker automation, SNS event routing, and cross-account S3/IAM infrastructure.
  • Ground-up rewrote the training stack in 2024 to PyTorch Lightning + Hydra + Optuna + W&B — primary author across 176 commits — adding ResNet, Transformer, and tsai time-series architectures plus a YOLO-based Valsalva-maneuver detector as a preprocessing step. Built for FDA submission auditability.
  • Shipped a serverless multi-modal ensemble inference framework in 2025 combining signal-only and signal+spectrogram models across folds. Deployed and handed off to Semler Scientific for production.

2021 → 2025 · ML Pipeline Engineer

TexNano — Amrita Medical AI

Lead engineer on the EUS_ML platform — endoscopic ultrasound machine learning — and the connected clinical labeling and visualization tools. ~$590K cumulative McGovern Institute funding (2023–2025). Models in clinical use today.

  • Engineered the full EUS video-to-dataset pipeline in 2021 — frame extraction, EUS-specific artifact filtering (doppler overlays, GUI elements, pointer markings), station-level label generation, and a GPU/CPU inference engine for automatic pancreatic cancer detection. The dataset foundation everything since builds on.
  • Shipped eusml-labeller — a Remix-based real-time web labeller used by clinicians during live endoscopic procedures, replacing retrospective video review. Added Tumor Timer and patient-data encryption as the platform matured.
  • Built the pancreas segmentation data pipeline (2024–2025) from first commit — video frame extraction, manifest generation, DynamoDB metadata integration, CVAT annotation integration. Tuned sampling strategy down from whole-video extraction to a 25-frame-per-video target.
  • Trained the VR pediatric heart segmentation model (2024–2025) from CT scans of pediatric congenital-heart-disease patients. Model is in clinical use as of 2025, feeding a VR surgical-planning visualization. Shipped a standalone inference service so clinicians could run it without touching the ML stack.
