Problem

Pediatric patients with congenital heart disease need careful surgical planning, and surgeons at the collaborating hospital were using VR to visualize the heart in 3D. The missing piece was automatic segmentation: someone had to manually outline the heart from CT scans before it could be loaded into the VR environment. a clinical collaborator needed an ML model to do this automatically.

Contribution

Architected and built the complete ML pipeline infrastructure using PyTorch Lightning and Hydra, transforming the codebase from prototype notebooks to a modular, configurable training system. Implemented distributed data parallel (DDP) training support, comprehensive experiment tracking via Weights & Biases, and robust model checkpointing. Debugged and resolved training issues across single-GPU and multi-GPU setups.

Technical details

Structured training around PyTorch Lightning's LightningModule, DataModule, and callback patterns. Hydra configuration layers decoupled hyperparameters, model architecture, optimizer settings, and data paths into composable YAML configs. Integrated MONAI's segmentation metrics (Dice, Hausdorff) with custom callbacks that log per-epoch results to W&B alongside segmentation visualizations.

Built DDP support by ensuring data preparation runs once at module init, synchronizing dict logging across workers, and restricting segmentation visualization logging to the rank-0 process to avoid duplicate entries. Implemented checkpoint management with best-model saving based on validation metrics.

The inference pipeline retrieves trained models from W&B checkpoints via the W&B API and runs predictions through Lightning's evaluation utilities. Dataset loading for ImageCHD and BRATS uses MONAI preprocessing with ResizeWithPadOrCrop transforms. Optimized dataset downloads to a shared mount point to prevent redundant I/O when multiple users run training.

Inference service (2025): Wrapped the trained model into a standalone service that clinicians can run directly — doctors upload a pediatric CT scan and receive a heart segmentation back, which feeds into the VR visualization pipeline without requiring any ML infrastructure knowledge.

Outcome

The segmentation model is in clinical use as of 2025 — surgeons can now automatically generate a heart segmentation from a pediatric CT scan and load it directly into the VR environment for pre-surgical planning. The training pipeline (Lightning + Hydra + MONAI + W&B) runs across multiple GPUs on the lab's shared infrastructure.
